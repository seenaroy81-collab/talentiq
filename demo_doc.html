02: THE TECH BRAIN

The Magic Engine.

We built this using the most modern "Lego bricks" available in the tech world. Here is how the parts work together:

The Front & Back

R
React + Vite (The Face)

Makes everything look beautiful and fast. Like a high-end luxury car dashboard.

N
Node.js + Express (The Brain)

The traffic controller that handles all data and talks to the AI.

The Memory Bank

M
MongoDB (The Long-Term Memory)

Where we store resumes, users, and every interview forever.

⚡
Redis (The Short-Term "Sticky Note")

Ultra-fast memory for things we need right now (like interview states).

The Genius (AI)

We use Groq—the world's fastest AI engine—to run Llama 3.3 (70B). This is many times smarter than a basic chatbot; it understands complex code and technical nuance.

Did you know?

Our system uses Whisper 3 to "listen" to the candidate's voice and turn it into text so the AI can understand it!

03: THE DELIVERY

Ready Anywhere.

The Docker Lunchbox 🍱

Imagine you made a perfect sandwich. If you just tell a friend the recipe, they might use bad bread. But if you put the whole sandwich in a Lunchbox, it stays perfect!

Docker = The Ultimate Lunchbox

It packs the Code + Database + Server into one box that runs EXACTLY the same on every computer.

Why do we use it?

  • No more "It works on my machine": If it works inside Docker, it works everywhere.
  • Instant Setup: Type one command, and the whole system (Database, Redis, etc.) starts up automatically.
  • Super Safe: Each part of the app lives in its own little world, isolated from the others.

Other Tools we used

🛡️ Clerk

Handles "Who can log in" so we don't have to build security from scratch.

☁️ Cloudinary

The "Cloud Garage" where all resume PDFs are safely stored.

📨 Inngest

The "Postman" that sends email reports to the hiring manager after the interview.

📺 Stream

The tech that powers 4K video calls and screen sharing.

04: VIVA SECRETS

Demo Mastery.

Preparing for questions from your team or teachers? Here is the cheat sheet!

Q1: Why use Redis when you already have MongoDB? A: MongoDB is great for storing big data, but Redis is 100x faster because it lives in "RAM". We use it for things that change fast, like interview status.
Q2: How does the AI know if my code is correct? A: We send your code to Llama 3.3. The AI understands the problem logic and explains exactly what's wrong—it's like having a Senior Engineer reviewing you!
Q3: What happens if the internet cuts out? A: Thanks to TanStack Query, the app handles "lag" gracefully, and we save progress in MongoDB constantly so nothing is lost.
Q4: Why Groq? Why not OpenAI? A: Groq is incredibly fast. It makes the AI response feel instant (sub-second), which is crucial for a real-time interview experience.
